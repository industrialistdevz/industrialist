## 🐛 Troubleshooting & Common Issues

### ❌ GUI Doesn't Appear After Execution

**Cause:** The script may have loaded before Roblox finished initializing.
**Fix:** Wait 5–10 seconds after your character spawns before executing. Also ensure your executor is properly attached.

---

### ❌ Auto Farm Isn't Collecting Resources

**Cause:** The `AutoFarm.radius` may be set too low, or the resource type isn't enabled.
**Fix:**

```bash
AutoFarm.radius = 250       -- increase radius
AutoFarm.enabled = true     -- confirm it's active
```

Also check: is your inventory full? Enable `AutoFarm.skipFullInventory = false` to override.

---

### ❌ Script Crashes on Injection

**Cause:** Version mismatch between the script and the current game build.
**Fix:** Pull the latest version from the repository:

```bash
git pull origin main
```

Then re-inject. If issues persist, check the Issues tab on GitHub for reported patches.

---

### ❌ Speed Boost Causes Teleportation Back (Rubber-Banding)

**Cause:** The server is correcting your position because the speed value is too high.
**Fix:** Reduce `Player.speedMultiplier` to `2.0` or lower. Values above `4.0` frequently trigger server-side correction.

---

### ❌ ESP Shows No Objects

**Cause:** ESP may be conflicting with a recent game update that changed object naming.
**Fix:** Toggle ESP off and back on from the GUI. If the problem persists, set:

```bash
ESP.enabled = false
-- Wait 10 seconds
ESP.enabled = true
```

---

### ❌ Mobile GUI Is Invisible on Android

**Cause:** System-level overlay permission is not granted.
**Fix:** Go to **Settings → Apps → [Executor App] → Display over other apps** and enable it. Restart both apps.
